1. Create `frontend/src/stores/features/git-web/GitWebStore.ts`:
   - `class GitWebStore` with `makeAutoObservable(this)` in constructor
   - Observable properties:
     - `currentRepo: GitRepo | null = null`
     - `currentBranch: string = ''`
     - `defaultBranch: string = ''`
     - `changedFiles: ChangedFile[] = []`
     - `commitMessage: string = ''`
     - `selectedFilePath: string | null = null`
     - `isLoading: boolean = false`
     - `isCommitting: boolean = false`
     - `error: string | null = null`
   - Computed properties:
     - `get stagedFiles(): ChangedFile[]` -- filter changedFiles where staged=true
     - `get unstagedFiles(): ChangedFile[]` -- filter changedFiles where staged=false
     - `get changedFileCount(): number` -- changedFiles.length
     - `get hasStagedFiles(): boolean` -- stagedFiles.length > 0
     - `get canCommit(): boolean` -- hasStagedFiles && commitMessage.trim().length > 0 && !isCommitting
   - Actions:
     - `setRepo(repo: GitRepo)` -- sets currentRepo, triggers loadStatus
     - `switchBranch(branch: string)` -- updates currentBranch, clears files/message/selection, triggers loadStatus
     - `stageFile(path: string)` -- sets staged=true on matching file
     - `unstageFile(path: string)` -- sets staged=false on matching file
     - `stageAll()` -- sets staged=true on all files
     - `unstageAll()` -- sets staged=false on all files
     - `setCommitMessage(message: string)` -- updates commitMessage
     - `selectFile(path: string | null)` -- updates selectedFilePath
     - `setChangedFiles(files: Omit<ChangedFile, 'staged'>[])` -- maps to ChangedFile with staged=false
     - `clearAfterCommit()` -- clears commitMessage, resets staged flags
     - `reset()` -- clears all state
   - The store does NOT make API calls directly -- that's done by TanStack Query hooks in the components. The store holds UI state only (staging, selection, commit message).
