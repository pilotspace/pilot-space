<objective>
Create backend API proxy router that fronts GitHub/GitLab operations, preventing direct browser-to-provider calls.

Purpose: All git operations go through the backend to avoid CORS issues, token exposure, and centralize rate limiting. Frontend calls /api/v1/git/... which proxies to the correct provider.

Output: Git proxy router with schemas, mounted in main.py, tested.
</objective>

1. Create `backend/src/pilot_space/api/v1/schemas/git_proxy.py` with Pydantic models:
   - `ChangedFileSchema(path: str, status: str, additions: int, deletions: int, patch: str | None)`
   - `GitStatusResponse(files: list[ChangedFileSchema], branch: str, total_files: int, truncated: bool)` -- truncated=True if >300 files (Pitfall 1)
   - `FileContentResponse(content: str, encoding: str, sha: str, size: int)`
   - `BranchSchema(name: str, sha: str, is_default: bool, is_protected: bool)`
   - `BranchListResponse(branches: list[BranchSchema], page: int, per_page: int)`
   - `CreateBranchRequest(name: str, from_ref: str)`
   - `FileChangeSchema(path: str, content: str, encoding: str = "utf-8", action: str = "update")`
   - `CommitRequest(branch: str, message: str, files: list[FileChangeSchema])` -- validate message non-empty, files non-empty
   - `CommitResponse(sha: str, html_url: str, message: str)`
   - `CreatePRRequest(title: str, body: str, head: str, base: str, draft: bool = False)`
   - `CreatePRResponse(number: int, html_url: str, title: str, draft: bool)`

2. Create `backend/src/pilot_space/api/v1/routers/git_proxy.py`:
   - `router = APIRouter(prefix="/git", tags=["git-proxy"])`
   - Helper: `async def _get_provider(session, current_user, integration_id: UUID) -> GitProvider` -- loads integration from DB, decrypts token, calls `resolve_provider(provider_type, token)`. Use the same pattern as `integrations.py` for fetching the integration record and decrypting the API key.
   - `GET /repos/{owner}/{repo}/status?branch={branch}&base_ref={base_ref}&integration_id={uuid}` -- returns GitStatusResponse with changed files. Include `truncated: True` warning if GitHub returns exactly 300 files.
   - `GET /repos/{owner}/{repo}/files/{path:path}?ref={ref}&integration_id={uuid}` -- returns FileContentResponse. Add 1MB size guard (Pitfall 7): if content exceeds 1MB, return 413 with "File too large".
   - `GET /repos/{owner}/{repo}/branches?search={search}&page={page}&per_page={per_page}&integration_id={uuid}` -- returns BranchListResponse
   - `POST /repos/{owner}/{repo}/branches?integration_id={uuid}` -- accepts CreateBranchRequest, returns BranchSchema
   - `DELETE /repos/{owner}/{repo}/branches/{branch_name}?integration_id={uuid}` -- returns 204
   - `GET /repos/{owner}/{repo}/default-branch?integration_id={uuid}` -- returns `{"default_branch": "main"}`
   - `POST /repos/{owner}/{repo}/commits?integration_id={uuid}` -- accepts CommitRequest, returns CommitResponse
   - `POST /repos/{owner}/{repo}/pulls?integration_id={uuid}` -- accepts CreatePRRequest, returns CreatePRResponse
   - Error handling: catch `GitHubRateLimitError`/`GitLabRateLimitError` and return 429 with Retry-After header. Catch `GitHubAuthError`/`GitLabAuthError` and return 401. Catch `GitHubAPIError`/`GitLabAPIError` and return 502 with provider error message.
   - All endpoints require `session: SessionDep, current_user: CurrentUser` (Gotcha 1: session ContextVar).
   - Use `""` not `"/"` for collection routes (Gotcha from CLAUDE.md FastAPI routing pattern).

3. Mount router in `backend/src/pilot_space/main.py`:

   - Add: `app.include_router(git_proxy_router, prefix=API_V1_PREFIX)` alongside other routers
