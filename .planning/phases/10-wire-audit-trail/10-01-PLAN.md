<objective>
Activate the three silent runtime audit wiring gaps identified by the v1.0 milestone audit:

1. **DI gap (AUDIT-01):** Add `audit_log_repository` as a `providers.Factory` in `container.py` and inject it into the 10 CRUD service factories that already guard with `if self._audit_repo is not None:`. Without injection, all CRUD audit writes are dead.
2. **RLS gap (AUDIT-01):** Import `set_rls_context` in `auth_sso.py` and call it before `write_audit_nonfatal` in `saml_callback` so the PostgreSQL RLS INSERT policy can resolve `app.current_user_id` and the audit row is actually persisted.
3. **AI session_factory gap (AUDIT-02 + AIGOV-03):** Pass `session_factory` to `PermissionAwareHookExecutor` in `pilotspace_agent.py` (via `_factories.py create_pilotspace_agent`) so `AuditLogHook._session_factory` is not None and the DB write path in `_create_audit_callback()` is reached for every AI tool invocation.

Purpose: These are 1–5 line wiring fixes. Service logic and DB schema are fully implemented. No new behavior required — only enabling existing audit paths.
Output: All three gap files patched + tests confirming wiring is active.
</objective>
