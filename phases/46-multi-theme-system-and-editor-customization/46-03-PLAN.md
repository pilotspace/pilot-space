  <action>
    Create `frontend/src/features/theme/tmtheme-parser.ts`:

    Export `parseTmTheme(xml: string): IStandaloneThemeData`:
    - Parse XML using DOMParser (browser-native, no extra dependency)
    - Extract `<dict>` entries from plist format:
      - First `<dict>` in settings array = global settings (background, foreground, caret, selection)
      - Subsequent entries = token rules with `scope` and `settings` (foreground, fontStyle)
    - Map plist color format (#RRGGBB or #RRGGBBAA) to Monaco 6-char hex (strip # and alpha)
    - Map scope strings to Monaco token names:
      - 'keyword.control', 'keyword' -> 'keyword'
      - 'string.quoted', 'string' -> 'string'
      - 'comment.line', 'comment' -> 'comment'
      - 'constant.numeric' -> 'number'
      - 'entity.name.type' -> 'type'
      - 'entity.name.function' -> 'function'
      - Pass through unmapped scopes as-is (Monaco handles gracefully)
    - Build colors object: map settings.background -> 'editor.background', settings.foreground -> 'editor.foreground', settings.caret -> 'editorCursor.foreground', settings.selection -> 'editor.selectionBackground'
    - Detect base theme: if background luminance > 128 -> 'vs', else if luminance > 20 -> 'vs-dark', else 'hc-black'

    Export `parseVSCodeTheme(json: string): IStandaloneThemeData`:
    - JSON.parse the input
    - Extract `type` field -> base ('vs', 'vs-dark', 'hc-black', 'hc-light')
    - Extract `colors` object -> map directly to Monaco colors (keys already match)
    - Extract `tokenColors` array -> map each to Monaco rule:
      - `scope` (string or string[]) -> split into individual rules
      - `settings.foreground` -> foreground (strip #)
      - `settings.fontStyle` -> fontStyle
    - Return `{ base, inherit: true, rules, colors }`

    Export `detectThemeBase(bgHex: string): 'vs' | 'vs-dark' | 'hc-black'`:
    - Calculate relative luminance from hex
    - Light (> 128) -> 'vs', dark (> 20) -> 'vs-dark', very dark -> 'hc-black'

    Update `frontend/src/features/theme/index.ts` to export parser functions.
  </action>
