  <action>
    1. Create `sdk/plugin-sdk-types.ts` — the types that define the host-plugin communication contract:
       - `PluginSDKMethod` = union of all callable methods: `'editor.getContent'`, `'editor.insertBlock'`, `'editor.replaceSelection'`, `'editor.registerBlockRenderer'`, `'commands.register'`, `'actions.register'`, `'ui.showToast'`, `'storage.get'`, `'storage.set'`
       - Map methods to required permissions:
         ```
         const METHOD_PERMISSIONS: Record<PluginSDKMethod, PluginPermission[]>
         editor.getContent -> ['editor:read']
         editor.insertBlock -> ['editor:write']
         editor.replaceSelection -> ['editor:write']
         editor.registerBlockRenderer -> ['editor:write']
         commands.register -> ['editor:write']
         actions.register -> ['editor:write']
         ui.showToast -> [] (no permission needed)
         storage.get -> ['storage:read']
         storage.set -> ['storage:write']
         ```
       - `PluginMessage` = `{ type: 'sdk-call'; id: string; method: PluginSDKMethod; args: unknown[] }`
       - `PluginResponse` = `{ type: 'sdk-response'; id: string; result?: unknown; error?: string }`
       - `PluginLifecycleEvent` = `{ type: 'lifecycle'; event: 'activate' | 'deactivate' }`
       - Export `METHOD_PERMISSIONS` map and all types

    2. Create `sandbox/message-protocol.ts` — host-side message handling:
       - `checkPermission(method: PluginSDKMethod, manifestPermissions: PluginPermission[]): boolean` — checks if plugin's declared permissions include all required permissions for the method
       - `createMessageHandler(pluginName: string, permissions: PluginPermission[], handlers: Record<PluginSDKMethod, (...args: unknown[]) => unknown | Promise<unknown>>)` — returns a `MessageEvent` handler that:
         a. Validates message origin (must match iframe)
         b. Checks permission via `checkPermission`
         c. Calls the appropriate handler
         d. Posts response back via `PluginResponse`
         e. Catches all errors and posts error response (never throws)
       - `sendLifecycleEvent(iframe: HTMLIFrameElement, event: 'activate' | 'deactivate'): void`

    3. Create `registry/PluginRegistry.ts` — runtime registry (plain module-level Map, not MobX):
       - `PluginInstance` = `{ name: string; iframe: HTMLIFrameElement; manifest: PluginManifest; registeredBlocks: string[]; registeredCommands: string[]; registeredActions: string[] }`
       - `registerPlugin(instance: PluginInstance): void`
       - `unregisterPlugin(name: string): void` — also cleans up registered blocks/commands/actions
       - `getPlugin(name: string): PluginInstance | undefined`
       - `getActivePlugins(): PluginInstance[]`
       - `getRegisteredBlockTypes(): Array<{ pluginName: string; type: string; label: string; icon: string }>` — aggregates all plugin-registered block types
       - `getRegisteredSlashCommands(): Array<{ pluginName: string; trigger: string; label: string; description: string }>` — aggregates plugin slash commands
       - `addBlockRegistration(pluginName: string, type: string): void`
       - `addCommandRegistration(pluginName: string, trigger: string): void`
       - `addActionRegistration(pluginName: string, actionId: string): void`
       - `clearAll(): void` — for testing
  </action>

  <action>
    1. Create `sandbox/PluginSandbox.tsx` — React component that renders a hidden iframe for one plugin:
       - Props: `{ manifest: PluginManifest; jsContent: string; onError?: (error: string) => void }`
       - Plain React component (NOT observer) — React 19 flushSync constraint
       - Renders `<iframe>` with `sandbox="allow-scripts"` (no allow-same-origin, no allow-forms)
       - Sets `srcdoc` to an HTML template that:
         a. Includes a `<script>` defining a `PilotPluginSDK` proxy object that translates method calls to postMessage SDK calls (e.g., `sdk.editor.getContent()` sends `{ type: 'sdk-call', method: 'editor.getContent', id: uuid, args: [] }`)
         b. Includes the plugin JS via inline `<script>{jsContent}</script>`
         c. Calls `onActivate(sdk)` if defined by the plugin
       - Registers a message event listener via `createMessageHandler` from message-protocol
       - The handler implementations:
         - `editor.getContent` — dispatches DOM CustomEvent `plugin:editor-get-content` and returns result
         - `editor.insertBlock` — dispatches `plugin:editor-insert-block` with `{ type, data }` detail
         - `editor.replaceSelection` — dispatches `plugin:editor-replace-selection` with `{ text }` detail
         - `editor.registerBlockRenderer` — calls `PluginRegistry.addBlockRegistration`, dispatches `plugin:block-registered`
         - `commands.register` — calls `PluginRegistry.addCommandRegistration`, dispatches `plugin:command-registered`
         - `actions.register` — calls `PluginRegistry.addActionRegistration` and calls `registerAction` from ActionRegistry with plugin-prefixed ID
         - `ui.showToast` — calls `toast()` from sonner
         - `storage.get` / `storage.set` — reads/writes `localStorage` under `plugin:{pluginName}:{key}`
       - On unmount: sends `deactivate` lifecycle event, unregisters from PluginRegistry, removes event listener
       - All errors caught in try/catch, passed to `onError` callback
       - Style: `display: none` (iframe is invisible)

    2. Create `hooks/usePluginLoader.ts`:
       - `usePluginLoader(workspaceId: string)` hook
       - Fetches enabled plugins from `/api/v1/workspaces/{workspaceId}/plugins/enabled`
       - For each enabled plugin, fetches JS content from its `storagePath` signed URL
       - Returns `{ plugins: Array<{ manifest: PluginManifest; jsContent: string }>, isLoading: boolean, error: string | null }`
       - Uses TanStack Query with `queryKey: ['workspace-plugins', workspaceId]` and `staleTime: 5 * 60 * 1000` (5 min)
       - Lazy-loads plugin bundles only when list is resolved (dependent query pattern)
  </action>
