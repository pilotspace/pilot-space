    1. Create `workspace_plugin_repository.py`:
       - `WorkspacePluginRepository` class with async methods:
         - `list_by_workspace(workspace_id: UUID) -> list[WorkspacePluginModel]` — all plugins for workspace
         - `get_enabled_by_workspace(workspace_id: UUID) -> list[WorkspacePluginModel]` — only status='enabled'
         - `get_by_id(plugin_id: UUID) -> WorkspacePluginModel | None`
         - `get_by_name(workspace_id: UUID, name: str) -> WorkspacePluginModel | None`
         - `create(model: WorkspacePluginModel) -> WorkspacePluginModel`
         - `update(model: WorkspacePluginModel) -> WorkspacePluginModel`
         - `delete(plugin_id: UUID) -> None`
       - Follow `artifact_annotation_repository.py` pattern with session injection

    2. Create `workspace_plugin_service.py`:
       - `WorkspacePluginService` with DI-injected repository
       - Methods:
         - `upload_plugin(workspace_id, manifest_dict, js_content_bytes) -> WorkspacePlugin` — validates manifest JSON against expected schema (name, version, entrypoint required), stores JS in Supabase Storage at `plugins/{workspace_id}/{name}/{version}/index.js`, creates DB record
         - `list_plugins(workspace_id) -> list[WorkspacePlugin]`
         - `get_enabled_plugins(workspace_id) -> list[WorkspacePlugin]`
         - `toggle_plugin(plugin_id, status: str) -> WorkspacePlugin` — sets status to 'enabled'/'disabled'
         - `delete_plugin(plugin_id) -> None` — removes from storage and DB
       - Manifest validation: require name (alphanumeric + hyphens), version (semver pattern r'^\d+\.\d+\.\d+$'), entrypoint

    3. Create `workspace_plugins.py` router:
       - Mount at `/api/v1/workspaces/{workspace_id}/plugins`
       - `GET ""` — list all plugins (any member)
       - `GET "/enabled"` — list enabled plugins only (any member)
       - `POST ""` — upload plugin (admin/owner only). Accepts multipart form: `manifest` (JSON string) + `bundle` (file upload, .js). Max 1MB bundle size.
       - `PATCH "/{plugin_id}/status"` — toggle enable/disable (admin/owner). Body: `{ "status": "enabled" | "disabled" }`
       - `DELETE "/{plugin_id}"` — delete plugin (admin/owner)
       - Use `SessionDep` in every route signature (per Gotcha #1)
       - Use `""` not `"/"` for collection routes (per FastAPI routing pattern in memory)

    4. Wire into DI container:
       - Add `workspace_plugin_repository` and `workspace_plugin_service` providers in `container.py`
       - Add `workspace_plugins` to `wiring_config.modules` list

    5. Mount router in `main.py`:
       - Import and include `workspace_plugins.router` with appropriate prefix
  </action>
