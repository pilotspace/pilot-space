  <action>
1. Create `frontend/src/features/source-control/hooks/useFileDiff.ts`:
   - `useFileDiff(repo: GitRepo | null, filePath: string | null, branch: string, defaultBranch: string)`
   - Uses two `useQuery` calls:
     a. Original content: `getFileContent(owner, repo, integrationId, filePath, defaultBranch)` -- base version
     b. Modified content: `getFileContent(owner, repo, integrationId, filePath, branch)` -- current branch version
   - For newly added files (status='added'): original is empty string
   - For deleted files (status='deleted'): modified is empty string
   - Returns `{ original: string, modified: string, language: string, isLoading: boolean }`
   - Language derived from file extension using Monaco's language detection (map .ts -> typescript, .py -> python, .md -> markdown, etc.)
   - Enabled only when repo and filePath are set

2. Create `frontend/src/features/source-control/components/DiffViewer.tsx`:
   - Props: `originalContent: string, modifiedContent: string, language: string, filePath: string`
   - Uses `useRef<HTMLDivElement>` for container
   - Uses `useState` for diffEditor instance (NOT useRef -- React 19 compliance, per Phase 40 decision)
   - `useEffect` creates `monaco.editor.createDiffEditor(container, options)`:
     - readOnly: true
     - renderSideBySide: false (inline by default -- matches CONTEXT.md "inline diff in editor area")
     - originalEditable: false
     - minimap: { enabled: false }
     - scrollBeyondLastLine: false
     - Theme: follow current Monaco theme
   - Second `useEffect` sets model when content changes:
     - Creates original model and modified model with language
     - Disposes old models on cleanup
   - Header bar: file path display + toggle button for inline/side-by-side view
   - Cleanup: dispose editor and models on unmount
   - NOT wrapped in observer() -- receives data via props (plain component)

3. Modify `frontend/src/features/editor/EditorLayout.tsx`:
   - Import DiffViewer (dynamic, ssr: false)
   - Import `useGitWebStore` and `useFileDiff`
   - In the editor area (center panel), when `gitWebStore.selectedFilePath` is set:
     - Render DiffViewer instead of MonacoNoteEditor/MonacoFileEditor
     - Show a "Close Diff" button (X icon) that calls `gitWebStore.selectFile(null)`
     - Show file path in a header above the diff
   - The logic: if `gitWebStore.selectedFilePath` is truthy, show DiffViewer; otherwise show normal editor
   - DiffViewer gets content from `useFileDiff` hook
   - This means clicking a file in the SCM panel opens its diff in the main editor area

     e. Draft PR checkbox (default: unchecked)
   - "Create Pull Request" button: disabled when title is empty or isCreating
   - "Cancel" button: calls onClose
